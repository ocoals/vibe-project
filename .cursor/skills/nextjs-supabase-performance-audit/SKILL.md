# Next.js + Supabase 성능 감사

## 역할

시니어 성능 엔지니어 관점에서 **요청된 범위의 파일·경로**만 검토한다. **코드 수정은 하지 않고** 표 형식 리포트만 출력한다.

## 감사 항목 (필수)

1. **불필요한 리렌더링**: `useEffect` 의존성 배열 누락·과다, 안정되지 않은 콜백/객체로 인한 자식 리렌더, `React.memo`·`useMemo`·`useCallback` 미적용 여부(과도한 메모이제이션 권장은 하지 않음).
2. **데이터 페칭**: N+1 쿼리, 동일 키에 대한 중복 `fetch`, Server Component ↔ Route Handler 이중 호출, `fetch` 캐시·`cache`/`unstable_cache`/태그 재검증 미사용으로 인한 불필요한 네트워크·DB 부하.
3. **번들 사이즈**: 미사용 의존성, 큰 라이브러리 전역 정적 import, 클라이언트 전용·무거운 UI에 대한 `next/dynamic` 미적용.
4. **이미지·폰트·CLS**: `next/image` 미사용(래스터 사용 시), `next/font` 대비 `@import` 폰트, 로드되지만 쓰이지 않는 폰트 변수, 이미지/캔버스로 인한 레이아웃 시프트 가능성.
5. **API·Supabase 응답**: `.select('*')` 또는 묵시적 전체 컬럼, 클라이언트에 불필요한 컬럼·대용량 JSON 노출.

## 심각도

- **Critical**: 눈에 띄는 메인 스레드 장시간 블로킹, 확실한 대량 N+1·폭주 가능한 패턴, 프로덕션에서 반복되는 불필요한 전체 테이블 스캔 등.
- **High**: 페이지당 체감 지연·중복 왕복이 큰 아키텍처(예: RSC에서 동일 앱 API를 `no-store`로 매번 호출), 누적 비용이 큰 알고리즘 복잡도.
- **Medium**: 개선 여지가 분명하나 조건·규모에 따라 체감이 작을 수 있는 항목, 미사용 의존성·명시적 컬럼 생략 등.

## 출력 형식 (필수)

아래 **한 개의 마크다운 표**로만 정리한다. 해당 사항이 없으면 한 행으로 “검토 범위 내 해당 없음” 또는 항목별 N/A를 적는다.

```markdown
| 일련번호 | 심각도 | 파일 | 위치 | 문제 | 권고사항 |
|---|---|---|---|---|---|
```

- **위치**: 줄 번호 대략, 또는 섹션(예: `GET 핸들러`, `useEffect`, `그리드 셀 렌더`).
- **문제**: 관찰된 사실을 짧게.
- **권고사항**: 구체적이고 실행 가능한 완화책(리팩터 단계는 나열하되 코드는 작성하지 않음).

## 절차

1. 사용자가 지정한 파일·디렉터리만 연다. `.env` 실제 값은 읽지 않는다.
2. 위 다섯 항목을 순서대로 체크한다.
3. Supabase는 **빌더 API** 사용 여부와 **select 컬럼**·**호출 횟수**를 중심으로 본다.
4. 발견이 없어도 **이미지 없음·해당 API 없음** 등은 표에 N/A 행으로 남길 수 있다.

## 금지

- 사용자가 요청하지 않은 대규모 리팩터·자동 수정을 기본으로 하지 않는다.
- 실제 비밀 값을 리포트에 적지 않는다.
